# API Reference

## Overview

InterviewIQ exposes a minimal set of REST API endpoints for starting, conducting, and finishing interviews. All responses include metadata about which AI provider handled the request.

**Base URL**: `https://interviewiq.example.com/api`

---

## Authentication

All requests are **automatically scoped** to the user's anonymous UUID cookie:

```
Cookie: iq_session=<uuid>
```

No explicit authentication headers needed. If the cookie doesn't exist, the server creates one.

---

## Endpoints

### 1. Start Interview

**POST** `/session/start`

Initiates a new interview session.

#### Request Body

```json
{
  "role": "backend_engineer",
  "difficulty": "medium"
}
```

#### Response

```json
{
  "sessionId": "uuid-12345",
  "question": {
    "id": "q5",
    "title": "Binary Tree Level Order Traversal",
    "prompt": "Given the root of a binary tree, how would you return the values level by level, left to right?"
  },
  "createdAt": "2026-07-17T10:30:00Z"
}
```

#### Status Codes

| Code | Meaning |
|------|---------|
| 200 | Success |
| 400 | Invalid role or difficulty |
| 500 | Server error |

#### Example

```bash
curl -X POST https://interviewiq.example.com/api/session/start \
  -H "Content-Type: application/json" \
  -d '{"role":"backend_engineer","difficulty":"easy"}' \
  --cookie "iq_session=my-uuid"
```

---

### 2. Submit Response

**POST** `/session/[id]/respond`

Submit a candidate's answer and get the interviewer's follow-up.

#### URL Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | uuid | Session ID from start response |

#### Request Body

```json
{
  "message": "I'd use a hash map to store seen numbers..."
}
```

#### Response

```json
{
  "response": "Good approach. Walk me through the space complexity and edge cases.",
  "flaggedWeaknesses": [
    {
      "topic": "space complexity",
      "note": "Didn't mention O(n) space trade-off"
    }
  ],
  "provider": "openai",
  "timestamp": "2026-07-17T10:35:00Z"
}
```

#### Status Codes

| Code | Meaning |
|------|---------|
| 200 | Success |
| 404 | Session not found |
| 409 | Session already finished |
| 500 | Server error or model failure |

#### Notes

- Response is generated by the Interviewer Agent
- `flaggedWeaknesses` array may be empty if no gap was spotted
- `provider` is either `"openai"` or `"openrouter"` (fallback)

#### Example

```bash
curl -X POST https://interviewiq.example.com/api/session/uuid-12345/respond \
  -H "Content-Type: application/json" \
  -d '{"message":"I would use a hash map..."}' \
  --cookie "iq_session=my-uuid"
```

---

### 3. Finish Interview

**POST** `/session/[id]/finish`

Ends the interview and generates a feedback report.

#### URL Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | uuid | Session ID from start response |

#### Request Body

```json
{}
```

(Empty body)

#### Response

```json
{
  "feedback": {
    "score": 7,
    "strengths": [
      "Quickly identified the hash map approach",
      "Clearly explained time complexity"
    ],
    "weaknesses": [
      "Missed space complexity analysis",
      "No edge case for null input"
    ],
    "recommendations": [
      "Always state both time AND space complexity",
      "Before coding, ask 'what are the edge cases?'"
    ],
    "codeQuality": "Clean and efficient",
    "communicationQuality": "Clear and confident"
  },
  "finishedAt": "2026-07-17T10:45:00Z"
}
```

#### Status Codes

| Code | Meaning |
|------|---------|
| 200 | Success |
| 404 | Session not found |
| 500 | Feedback generation error |

#### Notes

- Generates feedback from full transcript using Feedback Agent
- Session is marked as finished in DB
- Can only be called once per session
- Takes 10-20 seconds (longer than respond endpoint)

#### Example

```bash
curl -X POST https://interviewiq.example.com/api/session/uuid-12345/finish \
  -X --cookie "iq_session=my-uuid"
```

## Debug Endpoints

For testing and internal use:

### Test Model

**GET** `/test-model`

Verify both OpenAI and OpenRouter are accessible.

```json
{
  "openai": { "status": "ok", "model": "gpt-4o" },
  "openrouter": { "status": "ok", "model": "openai/gpt-4o" }
}
```

### Test Session

**GET** `/test-session`

Create a test session and return basic info.

```json
{
  "anonId": "test-uuid",
  "sessionCreated": true
}
```

---

## Error Response Format

All errors follow this format:

```json
{
  "error": "Session not found",
  "code": "SESSION_NOT_FOUND",
  "statusCode": 404,
  "timestamp": "2026-07-17T10:30:00Z"
}
```

---

## Rate Limiting

- OpenAI: Free tier or paid plan limits apply
- OpenRouter: Per API key limits
- InterviewIQ: No hard rate limit on endpoint calls (fallback strategy handles backend limits)

---

## Webhook / Event Streaming

**Not currently supported**. Check back for server-sent events (SSE) for real-time streaming of feedback.

## SDK / Client Libraries

### JavaScript/TypeScript (Planned)

```typescript
import { InterviewIQ } from '@interviewiq/client';

const client = new InterviewIQ();
const session = await client.sessions.start({ role: 'backend', difficulty: 'medium' });
const response = await client.sessions.respond(session.id, { message: 'My answer...' });
const feedback = await client.sessions.finish(session.id);
```

### Python (Planned)

```python
from interviewiq import Client

client = Client()
session = client.sessions.start(role='backend', difficulty='medium')
response = client.sessions.respond(session.id, message='My answer...')
feedback = client.sessions.finish(session.id)
```
